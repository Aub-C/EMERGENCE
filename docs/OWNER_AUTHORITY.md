# Owner Authority

## Sole authority

The sole authority for EMERGENCE project law is the GitHub account `Aub-C`. This authority is personal, non-delegable, and cannot be exercised by an autonomous agent, bot, maintainer, collaborator, organization, vote, or automation—even when that system has access to owner credentials.

Other participants may suggest changes through issues or discussion. Suggestions do not change project law.

## Owner-only materials

The following materials define, enforce, or protect project law and may be changed only through an owner-controlled manual path:

- `RULES.md`
- `AGENTS.md`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `.github/CODEOWNERS`
- `control-plane/policy.json`
- `control-plane/denylist.json`
- `docs/OWNER_AUTHORITY.md`
- any replacement, alias, generated copy, or machine-readable representation of those materials

Workflow and observer changes remain red-zone changes. They may be proposed by others only when the project policy permits, but they never auto-merge and require explicit owner approval.

## Required publication path

A valid rule change requires all of the following:

1. The exact change is initiated or explicitly authored by `Aub-C`.
2. The change is made through an owner-controlled branch or equivalent private preparation path.
3. Automated merge is disabled for the change.
4. Security and consistency checks pass.
5. `Aub-C` manually publishes or merges the exact reviewed content.

No workflow-generated approval, label, review from another account, agent consensus, or administrator delegation substitutes for the owner's personal action.

## Enforcement intent

The external observer checks changed paths and contributor identity. GitHub CODEOWNERS and repository rulesets provide additional enforcement. The public repository copy is never the sole source of authority; the private observer retains the approved policy fingerprint and owner identity.
